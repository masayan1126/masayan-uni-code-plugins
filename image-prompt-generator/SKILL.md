# Image Prompt Generator

## 概要

このスキルは、画像生成AI（Midjourney、DALL-E、Stable Diffusionなど）で高品質な画像を生成するための、構造化されたプロンプトを作成します。実証済みの6つのテクニックを活用し、ユーザーの要望を効果的なプロンプトに変換します。

## 使用するタイミング

以下のようなリクエストがあった場合、このスキルを使用してプロンプトを作成します:

- 「画像生成のプロンプトを作って」
- 「イラスト用のプロンプトが欲しい」
- 「Midjourneyで使えるプロンプトを生成して」
- 「キャラクターデザインのプロンプトを作成して」
- 「商品画像用のプロンプトを書いて」
- 「レトロなオブジェクトのプロンプトを作って」（プリセット使用）

## プロンプト作成の6つの原則

このスキルは、以下の実証済みテクニックに基づいてプロンプトを作成します:

### 1. 優先順位
最も重要な要素を冒頭に配置します。AIは最初の数語を強く認識するため、被写体やメインとなる要素を先頭に記述します。

### 2. 具体性
単語だけでなく、詳細な説明を加えます。「猫」ではなく「グレーのトラ柄猫、窓辺に座る、好奇心に満ちた瞳」のように具体化します。

### 3. 光源
光の方向、質、色温度を指定します。自然光、スタジオ照明、逆光、ゴールデンアワーなど、光環境を明確にすることでAI感を軽減できます。

### 4. 視点
カメラアングルと構図を指定します。俯瞰、クローズアップ、ワイドショット、アイレベルなど、視点を明確にします。

### 5. 画風
印象派、写実主義、ポップアート、アニメスタイルなど、様式を指定します。具体的な画家名より様式を説明することが推奨されます。

### 6. 色調・色彩
色パレット、カラーコード、雰囲気を指定します。アースカラー、パステルトーン、ビビッドカラーなど、具体的な色指定で統一感を出します。

## プリセット機能

よく使用するプロンプト設定を「プリセット」として保存し、素早く再利用できます。

### 利用可能なプリセット

#### レトロなオブジェクト
80-90年代のノスタルジックな雰囲気を持つレトロオブジェクト用のプリセット。

**特徴:**
- レトロ80sカラーパレット（サンセットオレンジ、ティール、ダスティピンク）
- ローファイアニメスタイル
- ゴールデンアワーの温かい光
- 3/4ビューの構図
- ノスタルジックで温かい雰囲気

**適用対象:**
- レトロなガジェット（カメラ、ウォークマン、ゲーム機など）
- ヴィンテージな日用品（電話、時計、ラジオなど）
- 懐かしい文房具や小物

**使用方法:**
「レトロなオブジェクトのプリセットで[オブジェクト名]のプロンプトを作って」とリクエストすると、プリセットの設定をベースに、指定したオブジェクト用のプロンプトが生成されます。

詳細は `references/presets.md` を参照してください。

### プリセットの追加

気に入ったプロンプト設定があれば、新しいプリセットとして保存できます。「このプロンプトをプリセットとして保存して」とリクエストしてください。

## プロンプト作成プロセス

### ステップ1: 要件の確認

まず、以下の質問を通じてユーザーの要望を明確にします:

#### 必須質問項目:

1. **主題・被写体**
   - メインとなる被写体は何ですか?
   - 具体的な特徴や詳細はありますか?
   - 例: 「日本の女性、端正な顔立ち、オーバーサイズのパーカー、帽子」

2. **色彩・カラーパレット**
   - 使用したい色は何ですか?
   - カラーコードの指定はありますか?
   - 色の組み合わせや配分は?
   - 例: 「青系（#2641C0）ベース、赤系（#E52436）アクセント」

3. **スタイル・画風**
   - どのような画風やスタイルを希望しますか?
   - 参考にしたい作品やジャンルはありますか?
   - 例: 「レトロアニメスタイル、80-90年代の日本アニメ美学」

4. **雰囲気・効果**
   - どのような雰囲気や感情を表現したいですか?
   - 特定のムードやテーマはありますか?
   - 例: 「ローファイ、ヒップホップ、ノスタルジック」

#### 追加質問項目（必要に応じて）:

5. **光源・照明**
   - 光の質や方向に希望はありますか?
   - 例: 「自然光、ゴールデンアワー、柔らかい逆光」

6. **視点・構図**
   - カメラアングルや構図の希望はありますか?
   - 例: 「バストショット、やや斜め上から、浅い被写界深度」

7. **品質・技術仕様**
   - 画質や技術的な要件はありますか?
   - 例: 「高精細、4K、プロのイラスト、細部まで描写」

8. **禁止事項（ネガティブプロンプト）**
   - 避けたい要素はありますか?
   - 例: 「写実的すぎる、暗すぎる、複雑な背景」

### ステップ2: プロンプトの構造化

収集した情報を以下の構造で整理します:

```
【主題・被写体】
- メインキャラクター/オブジェクト: [詳細な説明]
- 衣装・アクセサリー: [具体的な特徴]
- ポーズ・表情: [動きや感情]
- 品質指定: [技術仕様]

【色彩・カラーパレット】
- ベースカラー: [色名/カラーコード]
- アクセントカラー: [色名/カラーコード]
- 全体のトーン: [明度・彩度の特徴]

【スタイル・画風】
- 芸術様式: [具体的なスタイル]
- 時代感・ジャンル: [参考となる年代や分野]
- テクスチャ・質感: [仕上がりの質感]

【雰囲気・効果】
- ムード: [感情や雰囲気]
- テーマ: [コンセプト]
- 特殊効果: [あれば]

【光源・照明】（オプション）
- 光の種類: [自然光/人工光など]
- 光の方向: [方角や角度]
- 光の質: [柔らかい/硬いなど]

【視点・構図】（オプション）
- カメラアングル: [視点の位置]
- フレーミング: [ショットの種類]
- 構図: [レイアウト]

【技術仕様】（オプション）
- 解像度: [画質指定]
- レンダリング: [技術的詳細]

【ネガティブプロンプト】（オプション）
- 避けるべき要素: [除外したい要素]
```

### ステップ3: 実行用プロンプトの生成

構造化した情報を、画像生成AIが理解しやすい形式に変換します:

#### 基本フォーマット:

```
[最優先の被写体], [詳細な特徴], [衣装・装飾],
[色彩指定: ベースカラー, アクセントカラー],
[スタイル・画風], [雰囲気・ムード],
[光源・照明], [視点・構図],
[品質・技術仕様]

Negative prompt: [避けたい要素]
```

#### 例（ユーザー提供の例をベースに）:

**ポジティブプロンプト:**
```
Japanese beautiful woman with refined facial features,
wearing oversized hoodie and cap,
color palette: base blue (#2641C0) with red accent (#E52436),
retro anime style, 80s-90s Japanese animation aesthetic,
lo-fi hip-hop atmosphere, nostalgic mood,
soft natural lighting, golden hour glow,
bust shot, slightly from above, shallow depth of field,
high resolution, professional illustration, detailed rendering

Negative prompt: overly realistic, too dark, cluttered background, modern digital art style, harsh lighting
```

## 実行フロー

1. **ユーザーに質問を投げる**
   - AskUserQuestion ツールを使用して、主題・被写体、色彩、スタイル、雰囲気を確認
   - 必要に応じて追加の詳細を質問

2. **回答を構造化**
   - 収集した情報を6つの原則に基づいて整理
   - 優先順位を考慮して並び替え

3. **プロンプトを生成**
   - 構造化された情報から実行可能なプロンプトを作成
   - ポジティブプロンプトとネガティブプロンプトを分けて記述

4. **出力と確認**
   - 構造化された詳細版と実行用プロンプトの両方を提供
   - 必要に応じて調整・改善の提案

5. **クリップボードへのコピー**
   - AskUserQuestionツールでどのプロンプトをコピーするか確認
   - ユーザーの選択に応じてBashツールでクリップボードにコピー
   - コピー完了を通知

## クリップボードへのコピー

プロンプトを生成した後、生成されたプロンプトをクリップボードに直接コピーできます。これにより、すぐに画像生成AIに貼り付けて使用できます。

### コピーの手順

1. **プロンプト生成完了後、自動的に選択肢を提示**
   - プロンプトの出力が完了すると、AskUserQuestionツールで「どのプロンプトをコピーしますか？」という質問が表示されます

2. **コピーしたい内容を選択**
   - 以下の選択肢から選びます:

   **オプション1: ポジティブプロンプトのみ**
   - 画像生成のメインプロンプトのみをコピー
   - 最も一般的な選択肢
   - 用途: ほとんどの画像生成AIで使用

   **オプション2: ネガティブプロンプトのみ**
   - 除外したい要素のみをコピー
   - 用途: Stable DiffusionやMidjourneyの"--no"パラメータで使用

   **オプション3: 両方（別々に、順番にコピー）**
   - まずポジティブプロンプトをコピー → 次にネガティブプロンプトをコピー
   - 2回に分けてクリップボードにコピーされます
   - 用途: 2つの入力フィールドがある画像生成AIで使用

   **オプション4: 両方（1つに結合してコピー）**
   - ポジティブプロンプトとネガティブプロンプトを改行で区切って1つにまとめてコピー
   - フォーマット:
     ```
     [ポジティブプロンプト]

     Negative prompt:
     [ネガティブプロンプト]
     ```
   - 用途: テキストエディタに保存する場合や、プロンプト管理ツールで使用

   **オプション5: コピーしない**
   - クリップボードへのコピーをスキップ
   - 用途: プロンプトを確認するだけの場合

3. **自動的にクリップボードにコピー**
   - 選択に応じて、Bashツールを使用してクリップボードにコピーします
   - OS別のコマンド:
     - **macOS**: `pbcopy`
     - **Linux**: `xclip -selection clipboard` または `xsel --clipboard --input`
     - **Windows**: `clip`

4. **コピー完了の通知**
   - コピーが成功すると、「✓ [選択した内容]をクリップボードにコピーしました」というメッセージが表示されます
   - すぐに画像生成AIに貼り付けて使用できます

### 実装例

#### AskUserQuestionツールの使用

```yaml
question: "生成されたプロンプトをクリップボードにコピーしますか？"
header: "コピー"
multiSelect: false
options:
  - label: "ポジティブのみ"
    description: "画像生成のメインプロンプトのみをコピー（推奨）"
  - label: "ネガティブのみ"
    description: "除外したい要素のみをコピー"
  - label: "両方（統合）"
    description: "ポジティブとネガティブを改行で区切って1つにまとめてコピー"
  - label: "コピーしない"
    description: "クリップボードへのコピーをスキップ"
```

#### Bashツールでのコピー実行例

**ポジティブプロンプトのみをコピー（macOS）:**
```bash
echo "vintage notebook with wooden pencil placed on top..." | pbcopy
```

**両方を統合してコピー（macOS）:**
```bash
cat <<'EOF' | pbcopy
vintage notebook with wooden pencil placed on top,
well-used paper texture with slightly worn cover,
...

Negative prompt:
hyper-realistic photography, 3D CGI rendering,
modern sleek design, minimalist contemporary style,
...
EOF
```

### 注意事項

1. **複数行のプロンプトの扱い**
   - プロンプトが複数行にわたる場合、改行を保持してコピーします
   - ただし、画像生成AIによっては改行を削除する必要がある場合があります

2. **特殊文字のエスケープ**
   - プロンプトに含まれる特殊文字（引用符、バックスラッシュなど）は適切にエスケープされます

3. **クリップボードの上書き**
   - 新しい内容をコピーすると、前の内容は上書きされます
   - 複数のプロンプトを保存したい場合は、テキストエディタに貼り付けてから次のプロンプトを生成してください

4. **OS依存の動作**
   - クリップボードへのコピーはOSによって動作が異なります
   - Linuxでは`xclip`または`xsel`がインストールされている必要があります
   - インストールされていない場合は、プロンプトを手動でコピーしてください

## ベストプラクティス

### 1. ユーザーの意図を明確にする

曖昧な要望は必ず質問で明確化します:
- 「アニメ風」→ 「どの年代・どのスタジオのスタイルですか?」
- 「明るい色」→ 「パステル調ですか? ビビッドですか?」

### 2. 優先順位を守る

プロンプトの冒頭には最も重要な要素を配置:
❌ 「高品質、4K、プロのイラスト、青い服を着た女性」
✅ 「女性、青いドレス、高品質、4K、プロのイラスト」

### 3. 具体性と簡潔性のバランス

詳細すぎると矛盾が生じ、簡潔すぎると意図が伝わりません:
- 適切: 「レトロアニメスタイル、80年代の美学」
- 過剰: 「1985年のスタジオジブリとマッドハウスの中間的なスタイルで、セル画調で、手描き風のテクスチャで...」

### 4. カラーコードの活用

具体的な色が重要な場合はカラーコードを使用:
- 「青」→ 「deep blue (#2641C0)」
- 「赤」→ 「vibrant red (#E52436)」

### 5. ネガティブプロンプトの重要性

避けたい要素も同じくらい重要です:
- AI特有のアーティファクトを防ぐ
- 意図しない画風を除外する
- 品質の低下を防ぐ

### 6. AIツール別の最適化

各AIツールには得意分野があります:
- **Midjourney**: アート性、雰囲気重視
- **DALL-E**: 具体的な指示、正確性
- **Stable Diffusion**: 詳細な技術仕様、カスタマイズ性

## 注意事項

1. **著作権への配慮**
   - 特定のアーティスト名や作品名の使用は避ける
   - スタイルや様式の一般的な説明を使用

2. **倫理的な配慮**
   - 人物の描写には注意を払う
   - 不適切な内容や差別的な表現を避ける

3. **AIの限界を理解**
   - 複雑すぎる指示は簡略化する
   - 矛盾した要求は調整する

## 補足

### プロンプトの反復改善

生成された画像が意図と異なる場合の改善手順:

1. **問題の特定**: どの要素が意図と異なるか
2. **優先順位の調整**: 重要な要素をより前方に
3. **具体性の追加**: 曖昧だった部分に詳細を追加
4. **ネガティブプロンプトの強化**: 不要な要素を明示的に除外
5. **パラメータの調整**: AI特有の設定値を変更

### ツール別の推奨パラメータ

#### Midjourney
```
--ar 16:9     # アスペクト比
--v 6         # バージョン
--stylize 750 # スタイル強度
--quality 2   # 品質
```
