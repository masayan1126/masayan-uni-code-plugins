# 実行ワークフロー

## 実行手順

### 1. 原稿の受け取り

ユーザーから原稿を受け取る（テキスト直接入力またはファイルパス指定）

### 2. REFERENCE.mdの読み込み

Xのアルゴリズムの評価ポイントを確認

### 3. 原稿の分析

- 核となるメッセージ
- ターゲット読者
- 専門性や実績
- 関連するトレンドやハッシュタグ

### 4. 3パターンの生成

#### パターン1: 短文投稿（140字）
- インパクトのある1行目
- プロフィールクリックを誘導
- 完結した価値提供
- 末尾にハッシュタグ3〜5個
- 文体: masayanスタイルの口語調

#### パターン2: スレッド投稿
- 各ツイート140字程度
- 3〜5ツイートで構成
- 1つ目のタイトルに『』を使用
- 1つ目の最後にハッシュタグ
- ストーリー性を持たせる

#### パターン3: 長文投稿
- 1000〜3000字程度
- 段落・見出し・箇条書きを活用
- ハッシュタグは末尾

### 5. 追加提案

- **ハッシュタグ**: トレンドを考慮した3〜5個
- **画像・図解アイデア**: 視覚コンテンツの提案
- **投稿時間帯**: ターゲット層に合わせた最適な時間

### 6. ユーザーへの提示

3パターンを提示し、どれをコピーするか確認

### 7. クリップボードへのコピー

選択されたパターンをクリップボードにコピー
