# note記事最適化ガイドライン

このドキュメントは、noteの公式ヘルプセンターから収集したベストプラクティスをまとめたものです。

## もっとも大事なこと

noteで記事を書く上で最優先すべきことは：

1. **創作を楽しみ続けること**
2. **ずっと発表し続けること**

これらは、ページビュー数、収益、フォロワー数よりも重要です。継続することが何よりも大切です。

## タイトルの作り方

### 基本原則
- **文字数**: 25文字以内（リスト表示やX（Twitter）で省略されないため）
- **明確性**: 「伝えたい人に、伝えたいことが伝わるか」を考える
- **具体性**: 数字やエピソードを使って読者がイメージしやすくする

### タイトルの例
❌ 悪い例: 「プログラミングについて」
✅ 良い例: 「初心者が3ヶ月でWebアプリを作った5つのステップ」

### ポイント
- 結論やオチを含める
- 具体的なキーワードを使う
- 誇張や煽りを避ける
- 読者が検索しそうな言葉を入れる

## 見出し画像の重要性

### データに基づく事実
- 見出し画像を設定すると、ページビュー数やスキ数が**数十パーセント増加**する
- 記事の第一印象を決める重要な要素

### 推奨事項
- **必ず見出し画像を設定する**
- SNSシェア用には**2:1のアスペクト比**が推奨（TwitterやFacebookで最適表示）
- 記事内容に関連した画像を選ぶ
- 「みんなのフォトギャラリー」やCanvaを活用できる

## 本文の書き方

### 記事構成の基本

#### 1. 冒頭のリード文（最重要）
- **文字数**: 50文字程度
- **内容**: 記事の要点や目次
- **理由**: この長さならX（Twitter）でシェアされたときに省略されない

#### 2. 結論を冒頭に書く
- 読者が最後まで読んでくれるとは限らない
- 重要なメッセージは早めに伝える
- 「読み切れる記事」を目指す

#### 3. 1記事1テーマの原則
- 複数のテーマを混ぜると読者が混乱する
- 焦点を絞って深く掘り下げる
- 必要な情報を1記事に簡潔にまとめる

### モバイル最適化（超重要）

noteの記事は**主にスマートフォンで閲覧される**ため、モバイル表示を最優先に考える。

#### 改行と段落
- **PCで3〜4行ごとに改行**する
- スマホで見ると適度な長さの段落になる
- 開発者ツールでスマホ表示を確認する

#### 視認性
- 長い文章の塊は避ける
- 適度に見出しを入れる
- 箇条書きを活用する

## 長文記事の分割戦略

### いつ分割すべきか

以下の場合は、複数記事への分割を検討しましょう。

- 記事が6,000文字を超える
- 複数の異なるテーマが含まれている
- 読者が情報過多で疲れる可能性がある

### 分割のメリット

1. **1記事1テーマの原則を守れる**
2. **各記事が読みやすい長さになる**
3. **シリーズ化でフォロワーの継続的な関心を維持**
4. **SEO効果が高まる**（複数の検索クエリに対応）

### シリーズ記事の作り方

**各記事の最後に相互リンクを配置**

```markdown
**シリーズ記事**
- ✅ 【準備編】○○○（本記事）
- 【ポートフォリオ編】△△△
- 【転職活動編】□□□
```

## 信頼性を高める工夫

### 筆者の実績を冒頭に配置

記事の信頼性を高めるため、筆者の実績を冒頭に配置しましょう。

**例**

```markdown
## 筆者の実績

31歳、文系大学卒。

完全独学でWeb系自社開発企業に転職。

学習10ヶ月で転職成功。

現在はシニアエンジニアとして、新人教育や採用面接も担当。
```

### 「この記事で分かること」セクション

リード文の後に「この記事で分かること」を配置すると、読者の期待値を適切に設定できます。

## ハッシュタグの設定

### ハッシュタグの数

- **推奨: 5〜10個**
- 多すぎるとスパムっぽく見える
- 少なすぎると発見されにくい

### テーマ別の推奨ハッシュタグ

**エンジニア転職系**
- #エンジニア転職 #プログラミング学習 #独学 #未経験エンジニア #駆け出しエンジニア #Web系自社開発 #転職活動 #キャリアチェンジ

**技術記事系**
- #プログラミング #技術記事 #エンジニア #開発 #コーディング + 具体的な技術名（#Ruby #JavaScript など）

**学習ログ系**
- #プログラミング学習 #学習ログ #駆け出しエンジニア #初心者エンジニア #100DaysOfCode

- [ ] 1記事1テーマになっているか

## まとめ

noteで読まれる記事を書くための核心は：

1. **継続すること**が何より大切
2. **タイトルは25文字以内**で具体的に
3. **見出し画像は必須**（PVが数十%増加）
4. **冒頭50文字のリード文**で要点を伝える
5. **1記事1テーマ**で焦点を絞る
6. **スマホ最適化**（PC で3〜4行ごとに改行）
7. **結論を冒頭に**書く
8. **筆者の実績を冒頭に配置**して信頼性を高める
9. **長文は分割してシリーズ化**する
10. **ハッシュタグを5〜10個設定**して発見性を高める
