# WHY/WHAT/HOW設計原則

CLAUDE.md作成時の設計原則を定義します。

## 1. WHY

**定義**: プロジェクトが存在する根本的な理由。1年後も変わらない本質。

**含めるべき**:
- プロジェクトの目的（1-2文）
- 対象者
- 使用言語

**含めない**:
- 具体的な機能説明
- バージョン情報

## 2. WHAT

**定義**: プロジェクトに含まれるコンポーネントの一覧。

**記載方法**:
- 名前と1行説明のみ（テーブル形式）
- 詳細は該当ファイルへの参照

## 3. HOW

**定義**: 使い方・ガイドライン。

**記載方法**:
- 外部ファイルへのインポート参照のみ
- `@docs/PRINCIPLES.md` 形式

## 4. 判断フロー

```
その情報は...
├─ 1年後も変わらない？ → WHY
├─ 「何がある」の一覧？ → WHAT（参照のみ）
└─ 使い方・ガイドライン？ → HOW（外部インポート）
```
