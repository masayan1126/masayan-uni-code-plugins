# サンプルプロンプト集

> **注意**: 各プロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## 例1: 日常系4コマ「本番環境でテスト」

### プロンプト

```
【タイトル】本番環境でテスト（画像上部に大きく配置）

【形式・レイアウト】4コマ漫画、2x2グリッド配置、コマ間に枠線
【読み順】左上→右上→左下→右下（Z字）
【ジャンル・雰囲気】日常系ギャグ、ほのぼの
【画風】カラー、手書き風、シンプルな線画、パステル調、スケッチ風タッチ
【キャラ設定】添付画像のキャラクターを主人公として使用。後輩キャラは明るい表情の別キャラとして描く。
【背景設定】オフィスのデスク、モニター複数台、観葉植物

コマ1: 新機能のコードを書き終えて満足げな表情でモニターを見つめる主人公エンジニア、コーヒーカップを持っている
コマ2: 後ろから話しかける後輩「ステージング環境壊れてるんで本番で確認していいですか？」主人公は振り返って驚いた表情
コマ3: 冷や汗を流しながら「いいわけないだろ！」と叫ぶ主人公、両手を広げて制止するポーズ
コマ4: 「じゃあ僕がやりますね」とにっこり笑って本番にデプロイする後輩、白目をむいて崩れ落ちる主人公

セリフ（日本語）:
コマ1: （よし、完璧だ...）
コマ2: 「先輩、ステージング環境動かないんですけど、本番で確認してもいいですか？」
コマ3: 「ダメに決まってるだろ！！」
コマ4: 「大丈夫ですよ、すぐ終わりますから」「ま、待て...」

効果音:
コマ1: カタカタ
コマ3: ドン！
コマ4: ポチッ

【補足指示】
- タイトル「本番環境でテスト」を画像上部に配置
- 手書き風テイスト、スケッチ調のタッチ
- 柔らかい線で漫画らしい表現
- コマ4は緊迫感と諦念を表現
```

### Xポスト文面

```
本番環境でテストするやつ、一回はいるよね...

ステージングが壊れてるからって本番で確認しようとする後輩、止めても無駄でした。

#エンジニアあるある #4コマ漫画 #プログラマー #IT業界 #開発者の日常 #AIイラスト
```

## 例2: バトル系縦スクロール「深夜のデプロイ」

### プロンプト

```
【タイトル】深夜のデプロイ（画像上部に大きく配置）

【形式・レイアウト】縦スクロール漫画、5コマ縦配置、フルワイドコマ
【読み順】上から下へ
【ジャンル・雰囲気】アクション、ドラマチック、緊迫感
【画風】カラー、手書き風、ダイナミックな線、スピード線、コントラスト強め
【キャラ設定】添付画像のキャラクターを主人公として使用。疲れ切った表情、ボサボサの髪、クマのある目で描く。
【背景設定】深夜のオフィス、暗い室内、モニターの光だけが照らす

コマ1: 深夜3時、誰もいないオフィスでモニターに向かうエンジニア、時計が3:00を指している、アングルは後ろから
コマ2: 「デプロイ開始」のボタンを押す瞬間、指のクローズアップ、緊張感
コマ3: プログレスバーが進む画面、エンジニアは祈るような表情、スピード線
コマ4: 「ERROR」の赤い文字がモニターに表示される、エンジニアの絶望した表情、衝撃波エフェクト
コマ5: 朝日が昇るオフィスの窓、机に突っ伏すエンジニア、空のエナジードリンク缶が散乱

セリフ（日本語）:
コマ1: （今日こそ...今日こそ終わらせる...）
コマ2: 「デプロイ...実行」
コマ3: （頼む...頼むから...）
コマ4: 「嘘だろ...」
コマ5: ...

効果音:
コマ2: ポチッ
コマ3: ジジジジジ...
コマ4: バァァァン！！
コマ5: ピヨピヨ（小鳥の声）

【補足指示】
- タイトル「深夜のデプロイ」を画像上部に配置
- 手書き風テイスト、スケッチ調のタッチ
- 緊迫感とドラマチックな演出を強調
- コマ4は特に衝撃的に
```

### Xポスト文面

```
深夜3時のデプロイ、祈りは届かなかった...

朝日と共に心が折れる音が聞こえた。

#エンジニアあるある #深夜デプロイ #本番リリース #プログラマー #IT業界 #開発者の日常 #AIイラスト
```

## 例3: キャラ固定ギャグ「動くんですけど...」

### プロンプト

```
【タイトル】動くんですけど...（画像上部に大きく配置）

【形式・レイアウト】3コマ、縦配置、表情にフォーカス
【読み順】上から下へ
【ジャンル・雰囲気】ギャグ、一発ネタ
【画風】カラー、手書き風、シンプル、デフォルメ、表情豊か
【キャラ設定】添付画像のキャラクターのみ登場（同一キャラ全コマ共通）
【背景設定】シンプルな白背景、デスクとモニターのみ

コマ1: コードレビューのコメントを見て困惑した表情、吹き出しに「なぜ動くか説明してください」
コマ2: 顎に手を当てて考え込む表情、汗が一滴
コマ3: 開き直ったようなにっこり笑顔で両手を広げる、「動くから動くんです！」

セリフ（日本語）:
コマ1: レビュアー「このコード、なぜ動くか説明してください」
コマ2: （...なぜだろう）
コマ3: 「動くから動くんです！」

効果音:
コマ2: シーン...
コマ3: キラーン☆

【補足指示】
- タイトル「動くんですけど...」を画像上部に配置
- 手書き風テイスト、スケッチ調のタッチ
- 表情の変化を強調
- コマ3は開き直った爽やかさを表現
```

## 例4: 日常系4コマ「レビューお願いします（3ヶ月放置）」

### プロンプト

```
【タイトル】レビューお願いします（画像上部に大きく配置）

【形式・レイアウト】4コマ漫画、2x2グリッド配置、コマ間に枠線
【読み順】左上→右上→左下→右下（Z字）
【ジャンル・雰囲気】日常系ギャグ、ほのぼの（でもちょっと闇）
【画風】カラー、手書き風、シンプルな線画、パステル調、スケッチ風タッチ
【キャラ設定】添付画像のキャラクターを主人公として使用
【背景設定】オフィスのデスク、カレンダー（日付が変わっていく演出）

コマ1: PRを作成して満足げな表情、吹き出しに「レビューお願いします！」日付は1月
コマ2: 1ヶ月後（2月のカレンダー）、「レビューお願いします〜」少し不安げな表情
コマ3: 3ヶ月後（4月のカレンダー）、枯れた目で「レビュー...お願いします...」植物が枯れている
コマ4: コンフリクト地獄の画面を見て白目、「マージできなくなった」

セリフ（日本語）:
コマ1: 「レビューお願いします！」（希望に満ちた表情）
コマ2: 「レビューお願いしますー...」（そろそろ見てくれないかな）
コマ3: 「レビュー...お願い...します...」（もはや祈り）
コマ4: 「コンフリクトが...止まらない...」

効果音:
コマ4: ドドドドド

【補足指示】
- タイトル「レビューお願いします」を画像上部に配置
- 手書き風テイスト、スケッチ調のタッチ
- 時間経過と共にキャラの疲弊を表現
- カレンダーで月日の経過を明示
```

### Xポスト文面

```
レビューお願いします（n回目）

3ヶ月放置されたPR、コンフリクトで死んだ。

#エンジニアあるある #コードレビュー #PR放置 #プログラマー #IT業界 #開発者の日常 #AIイラスト
```
