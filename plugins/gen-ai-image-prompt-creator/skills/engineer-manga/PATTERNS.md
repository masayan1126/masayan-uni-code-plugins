# 漫画パターン定義

## パターン1: 日常系4コマ（デフォルト）

| 項目 | 設定 |
|------|-----|
| 構成 | 起承転結の4コマ |
| レイアウト | 2x2グリッド配置、コマ間に枠線 |
| 読み順 | 左上→右上→左下→右下（Z字） |
| 画風 | カラー、手書き風、シンプルな線画、パステル調 |
| ジャンル | 日常系ギャグ、ほのぼの |
| 適したテーマ | 日常の小さな発見、ほっこりするオチ、共感系 |

### コマ構成ガイド（起承転結の黄金則）
- **起（1コマ目）**: 状況設定、普通の日常シーン
  - カメラ: **ワイド**推奨（全体を見せる）
- **承（2コマ目）**: 展開、何か起こる/気づく
  - カメラ: **ミディアム**推奨（会話・反応）
- **転（3コマ目）**: 予想外の展開、ツッコミポイント
  - カメラ: **クローズアップ**推奨（表情強調）
- **結（4コマ目）**: オチ、共感を誘う結末
  - カメラ: **大アップ or 引き**（インパクトの対比）

## パターン2: バトル系縦スクロール

| 項目 | 設定 |
|------|-----|
| 構成 | 4-6コマ縦配置 |
| レイアウト | 縦長スクロール形式、フルワイドコマ |
| 読み順 | 上から下へ |
| 画風 | カラー、ダイナミック、スピード線、コントラスト強め |
| ジャンル | アクション、バトル、ドラマチック |
| 適したテーマ | 締め切り、バグとの戦い、本番障害対応、修羅場 |

### コマ構成ガイド
- **導入（1-2コマ）**: 緊迫した状況の提示
- **展開（3-4コマ）**: 激しいアクション、連続的な動き
- **クライマックス（5-6コマ）**: 決着、衝撃的なオチ

### 効果音の例
- 「カタカタカタカタ！！」（激しいタイピング）
- 「ドン！！」（サーバーダウン）
- 「ピコーン！」（アラート）

## パターン3: キャラ固定ギャグ

| 項目 | 設定 |
|------|-----|
| 構成 | 2-4コマ自由形式 |
| レイアウト | 柔軟な配置、表情にフォーカス |
| 読み順 | 標準漫画順 |
| 画風 | カラー、シンプル、デフォルメ、表情豊か |
| ジャンル | ギャグ、一発ネタ |
| 適したテーマ | 一発ネタ、反応芸、キャラの個性を活かした笑い |

### キャラ設定例
- **疲れたエンジニア**: パーカー、クマ、コーヒー常備
- **元気な新人**: 希望に満ちた目、メモ帳持ち
- **無表情なベテラン**: すべてを悟った顔

---

## パターン選択の判断基準

| テーマ | 推奨パターン |
|-------|------------|
| 日常の小さなあるある | 日常系4コマ |
| ほっこり系 | 日常系4コマ |
| 締め切り・緊急対応 | バトル系縦 |
| 障害対応・修羅場 | バトル系縦 |
| シンプルな一発ネタ | キャラ固定ギャグ |
| リアクション系 | キャラ固定ギャグ |

---

## 共通設定

### キャラクター基本設定
- 20-30代のエンジニア
- カジュアルな服装（パーカー、Tシャツ）
- メガネ着用率高め
- 表情は豊かに

### 背景設定
- オフィス（デスク、モニター複数台）
- 会議室
- 自宅リモートワーク環境
- カフェ

### 手書き風テイストの指示
必ず以下を含める：
- 「手書き風の線画」
- 「スケッチ調のタッチ」
- 「柔らかい線」
- 「漫画らしい表現」

---

## 上級オプション: ラフ画（ネーム）からの清書

ユーザーがラフ画を持っている場合、構図や演出の意図を100%反映させる究極のテクニック。

### 使用方法
1. 紙とペンで棒人間レベルの簡単なコマ割り・キャラ配置を描く
2. ラフ画を写真に撮ってアップロード
3. 以下の指示をプロンプトに追加

### 追加指示文
```
【レイアウト参照】
アップロードしたラフ（下書き）のコマ割り・構図・キャラ配置に忠実に従って
漫画ページを仕上げてください。ラフの配置やバランスを正確に再現しつつ、
線画として清書するように描いてください。
```

### ポイント
- 画力は不要、重要なのは「何をどこに配置するか」の設計図
- クリエイターの演出意図をAIに正確に伝達できる
- 偶然ではなく意図通りの構図を実現
