# 図解スタイル定義

## 画像テイスト

### クリーン（デフォルト）
| 項目 | 設定 |
|------|-----|
| 線 | 整った直線・曲線、均一な太さ |
| 形状 | 幾何学的、正確な図形 |
| 質感 | 滑らか、デジタル |
| 雰囲気 | プロフェッショナル、モダン |
| プロンプト指示 | 「クリーンでモダンなインフォグラフィック、整った線、デジタルイラスト風」 |

### 手書き風
| 項目 | 設定 |
|------|-----|
| 線 | ラフな手描き線、不均一な太さ、少しブレのある線 |
| 形状 | 有機的、完璧でない図形 |
| 質感 | 紙に描いたような質感、鉛筆やマーカーのタッチ |
| 雰囲気 | 親しみやすい、カジュアル、ホワイトボード風 |
| プロンプト指示 | 「手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気」 |

## 図解タイプ

### 1. フローチャート型
| 項目 | 設定 |
|------|-----|
| 用途 | プロセス・手順の説明 |
| レイアウト | 左→右 または 上→下 の流れ |
| 要素 | ボックス、矢印、分岐 |
| 適したテーマ | CI/CDパイプライン、データ処理フロー、ユーザー認証フロー |

### 2. 比較表型
| 項目 | 設定 |
|------|-----|
| 用途 | 技術・ツールの比較 |
| レイアウト | 2-3列の並列配置 |
| 要素 | カード、チェックマーク、バッジ |
| 適したテーマ | フレームワーク比較、言語比較、サービス比較 |

### 3. 概念図型
| 項目 | 設定 |
|------|-----|
| 用途 | 抽象概念の視覚化 |
| レイアウト | 中心から放射状 または 階層構造 |
| 要素 | アイコン、接続線、ラベル |
| 適したテーマ | LLMの仕組み、機械学習の概念、設計パターン |

### 4. アーキテクチャ図型
| 項目 | 設定 |
|------|-----|
| 用途 | システム構成の説明 |
| レイアウト | レイヤー構造、ブロック配置 |
| 要素 | サーバー、データベース、API、矢印 |
| 適したテーマ | マイクロサービス構成、クラウドアーキテクチャ、RAG構成 |

### 5. タイムライン型
| 項目 | 設定 |
|------|-----|
| 用途 | 歴史・進化の説明 |
| レイアウト | 横軸に時間、イベントを配置 |
| 要素 | 年表、マイルストーン、アイコン |
| 適したテーマ | AI技術の歴史、プログラミング言語の進化、製品ロードマップ |

## 配色パターン

### テック系（デフォルト）
- メインカラー: 青 (#3B82F6)
- アクセント: 紫 (#8B5CF6)
- 背景: ダークグレー (#1F2937) または 白
- テキスト: 白 または 黒

### AI・機械学習系
- メインカラー: 青 (#0EA5E9)
- アクセント: 緑 (#22C55E), オレンジ (#F97316)
- 背景: 濃紺 (#0F172A)
- テキスト: 白

### 親しみやすい系
- メインカラー: パステルブルー
- アクセント: パステルピンク、パステルイエロー
- 背景: 白 または 薄いグレー
- テキスト: ダークグレー

### モノクロ系
- メインカラー: 黒
- アクセント: グレー
- 背景: 白
- テキスト: 黒

## レイアウト原則

### 基本ルール
1. **情報の流れは明確に**: 左→右 または 上→下
2. **要素は3-5個に絞る**: 情報過多を避ける
3. **余白を十分に**: 読みやすさを確保
4. **アイコンを効果的に**: 視覚的理解を助ける
5. **タイトルは上部に大きく**: 一目で内容が分かる

### アスペクト比
- 正方形 (1:1): SNS投稿用
- 横長 (16:9): プレゼン資料用
- 縦長 (9:16): ストーリーズ用

## アイコン・シンボル例

### 技術系
- サーバー: 四角いボックス
- データベース: 円筒
- API: 歯車または接続アイコン
- クラウド: 雲形
- ユーザー: 人型アイコン

### AI系
- ニューラルネットワーク: 接続されたノード
- データ: 積み重なったブロック
- 学習: 脳アイコン
- 出力: 矢印または電球
