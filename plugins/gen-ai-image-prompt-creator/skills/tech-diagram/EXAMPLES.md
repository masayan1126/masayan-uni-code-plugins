# サンプルプロンプト集

> **注意**: 各プロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## 例1: 概念図型「LLMの仕組み」

### プロンプト

```
【タイトル】LLMの仕組み（画像上部に大きく配置）

【図解タイプ】概念図
【カラースキーム】青・緑・オレンジのアクセント、濃紺背景
【レイアウト】左から右への流れ、3つの主要ステップ

【解説キャラクター】添付画像のキャラクターを右下に配置、図解を指し示すポーズ

【構成要素】
- 入力: ユーザーのテキスト入力
- 処理: トークン化 → エンベディング → Transformer → 出力生成
- 出力: 生成されたテキスト

【テキスト内容】
入力層: 「テキスト入力」
処理1: 「トークン化」 - テキストを小さな単位に分割
処理2: 「エンベディング」 - トークンを数値ベクトルに変換
処理3: 「Transformer」 - 自己注意機構で文脈を理解
出力層: 「テキスト生成」

【スタイル指示】
- インフォグラフィック風
- 日本語テキスト
- 各処理ステップを丸いノードで表現
- データの流れを矢印で明示
- ニューラルネットワークを連想させるデザイン
- キャラクターが解説している構図
```

### Xポスト文面

```
LLMの仕組みを図解してみた

入力→トークン化→エンベディング→Transformer→出力

この流れを理解するだけで、プロンプトエンジニアリングの考え方が変わります。

#LLM #AI解説 #テック図解 #ChatGPT #機械学習 #AIイラスト
```

## 例2: アーキテクチャ図型「RAGアーキテクチャ」

### プロンプト

```
【タイトル】RAGアーキテクチャ（画像上部に大きく配置）

【図解タイプ】アーキテクチャ図
【カラースキーム】青・紫グラデーション、ダークテーマ

- ユーザークエリ（左端）
- 検索層: ベクトルDB検索
- 生成層: LLMによる回答生成
- 出力（右端）

【テキスト内容】
ユーザー: 「質問を入力」
検索層:
- 「クエリをエンベディング」
- 「ベクトルDBで類似検索」
- 「関連ドキュメントを取得」
生成層:
- 「コンテキスト + クエリをLLMに入力」
- 「根拠のある回答を生成」
出力: 「回答を返却」

【スタイル指示】
- インフォグラフィック風
- 日本語テキスト
- データベースアイコン、AIアイコンを使用
- データフローを明確な矢印で表現
- 検索と生成の2つのフェーズを色分け
- キャラクターが解説している構図
```


## 例3: 比較表型「React vs Vue vs Svelte」

### プロンプト

```
【タイトル】React vs Vue vs Svelte（画像上部に大きく配置）

【図解タイプ】比較表
【カラースキーム】各フレームワークのブランドカラー（React:水色、Vue:緑、Svelte:オレンジ）、白背景
【レイアウト】3列の並列カード配置

【解説キャラクター】添付画像のキャラクターを下部中央に配置、両手を広げて比較を示すポーズ

【構成要素】
- React（左）
- Vue（中央）
- Svelte（右）
- 比較軸: 学習コスト、パフォーマンス、エコシステム

【テキスト内容】
React:
- 学習コスト: 「中程度」
- パフォーマンス: 「仮想DOM」
- エコシステム: 「最大規模」
- 特徴: 「JSX、Hooks」

Vue:
- 学習コスト: 「低め」
- パフォーマンス: 「仮想DOM」
- エコシステム: 「充実」
- 特徴: 「SFC、Composition API」

Svelte:
- 学習コスト: 「低め」
- パフォーマンス: 「コンパイル時最適化」
- エコシステム: 「成長中」
- 特徴: 「No Virtual DOM」

【スタイル指示】
- インフォグラフィック風
- 日本語テキスト
- 各フレームワークのロゴまたはアイコンを上部に配置
- 違いが一目で分かる配色
- カード形式で区切る
- キャラクターが解説している構図
```

```

#React #Vue #Svelte #フロントエンド #テック図解 #AIイラスト
```
