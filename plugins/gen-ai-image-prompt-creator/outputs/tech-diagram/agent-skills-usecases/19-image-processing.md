# 画像処理・変換 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】画像処理・変換（画像上部に大きく配置）

【図解タイプ】フローチャート（左から右への流れ）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青・緑・紫のマーカー、白またはクリーム色の背景
【レイアウト】入力画像→処理→出力の流れ

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、処理フローを指し示しながら解説している構図

【構成要素】
- 左側: 入力画像（複数枚）
- 中央: Agent Skill + スクリプト（6つの処理）
- 右側: 最適化された出力画像
- 下部: スクリプト活用のメリット

【テキスト内容】
左側（青）「入力画像」:
  画像アイコン複数
  - 大きなJPEG
  - 様々なサイズ
  - メタデータ付き

中央:
  「Agent Skill」
  「+ スクリプト」

  6つの処理（縦に並べる）:
  1. 「リサイズ」（800x600に）
  2. 「フォーマット変換」（WebP/AVIF）
  3. 「最適化・圧縮」
  4. 「サムネイル生成」
  5. 「メタデータ削除」
  6. 「一括処理」

右側（緑）「出力画像」:
  軽量化された画像アイコン
  - 最適化済み
  - Web用に軽量
  - プライバシー保護

下部のメッセージ:
  「スクリプトで決定論的処理」
  - トークン消費最小化
  - 高速な一括処理
  - 再現性・自動化が容易

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 6つの処理を縦に並べて
- 入力→出力のサイズ差を視覚的に
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「フローチャート型」を採用：入力→処理→出力の変換フロー
- 6つの処理を網羅的に表示

### コンセプト
- スクリプトによる決定論的処理
- LLMに画像を渡さずトークン節約
- ImageMagick/Sharpなどのツール活用
