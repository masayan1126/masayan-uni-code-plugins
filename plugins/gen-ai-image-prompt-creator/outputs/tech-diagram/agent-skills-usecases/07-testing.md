# テスト計画・テストコード生成 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】テスト計画・テストコード生成（画像上部に大きく配置）

【図解タイプ】概念図（ピラミッド構造）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】緑（Unit）・青（Integration）・紫（E2E）のグラデーション、白またはクリーム色の背景
【レイアウト】テストピラミッドを中央に配置、左右に補足情報

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、ピラミッドを指し示しながら解説している構図

【構成要素】
- 中央: テストピラミッド（3層）
  - 頂点（小さい）: E2E（紫）
  - 中間（中くらい）: Integration（青）
  - 底辺（大きい）: Unit（緑）
- 左側: テスト設計技法
- 右側: テストの特徴（速度・コスト・信頼性）
- 下部: テストの進め方

【テキスト内容】
ピラミッド:
  頂点「E2E」:
    - 少数
    - 遅い
    - 高コスト

  中間「Integration」:
    - 適度
    - 中速
    - 中コスト

  底辺「Unit」:
    - 多数
    - 高速
    - 低コスト

左側（テスト設計技法）:
  「テスト設計技法」
  - 境界値分析
  - 同値分割
  - デシジョンテーブル
  - 状態遷移

右側（テストの特徴）:
  速度: 「Unit > Integration > E2E」
  コスト: 「Unit < Integration < E2E」
  信頼性: 「適切なバランスが重要」

下部:
  「カバレッジ分析で漏れを特定」
  「テストケースを自動生成」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- ピラミッドは下が大きく上が小さい
- 各層を色分け（緑→青→紫）
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「概念図型（ピラミッド）」を採用：テストピラミッドは定番の可視化
- 各層の特徴（速度・コスト・量）を対比

### コンセプト
- テストピラミッドでバランスの重要性を伝える
- テスト設計技法を併記して実践的に
- Agent Skillでテストケース洗い出しからコード生成まで
