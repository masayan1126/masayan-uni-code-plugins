# デバッグ・調査 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】デバッグ・調査（画像上部に大きく配置）

【図解タイプ】フローチャート（循環型）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青・オレンジ・緑のマーカー、白またはクリーム色の背景
【レイアウト】4ステップの循環フロー、中央にゴール

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、デバッグフローを指し示しながら解説している構図

【構成要素】
- ステップ1（上）: エラー発生・スタックトレース（虫アイコン）
- ステップ2（右）: 仮説立案（電球アイコン）
- ステップ3（下）: 検証・絞り込み（虫眼鏡アイコン）
- ステップ4（左）: 根本原因特定（ターゲットアイコン）
- 中央: 「問題解決」
- ステップ間を矢印で循環接続

【テキスト内容】
ステップ1（青）「エラー発生」:
  - スタックトレース解析
  - エラーメッセージ解釈
  - 再現手順の確立

ステップ2（オレンジ）「仮説立案」:
  - 原因の仮説を複数列挙
  - 可能性の高い順に優先度付け
  - 検証方法を決定

ステップ3（緑）「検証・絞り込み」:
  - git bisect
  - 二分探索
  - ログ解析

ステップ4（紫）「根本原因特定」:
  - なぜなぜ分析
  - 対症療法ではなく根本解決
  - 再発防止策

中央:
  「問題解決」
  ターゲットアイコン

下部のメッセージ:
  「推測で修正しない。体系的に調査する」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 4ステップを循環型で配置
- 各ステップを色分け
- 中央に「問題解決」のゴールを配置
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「フローチャート型（循環）」を採用：調査は仮説→検証の繰り返し
- 中央にゴール（問題解決）を配置

### コンセプト
- 「推測で修正しない」がキーメッセージ
- 体系的な調査手法（仮説→検証→絞り込み）
- 根本原因分析で再発防止
