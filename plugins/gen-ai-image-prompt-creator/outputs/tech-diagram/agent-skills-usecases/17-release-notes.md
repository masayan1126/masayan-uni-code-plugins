# リリースノート生成 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】リリースノート生成（画像上部に大きく配置）

【図解タイプ】フローチャート（左から右への流れ）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青・緑・オレンジのマーカー、白またはクリーム色の背景
【レイアウト】コミットからリリースノートへの変換フロー

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、生成フローを指し示しながら解説している構図

【構成要素】
- 左側: 複数リポジトリのコミット
- 中央: Agent Skill（分析・グルーピング）
- 右側: 統合リリースノート
- 下部: リリースノートの構成

【テキスト内容】
左側（青）「コミット」:
  複数のリポジトリアイコン
  - repo-frontend
  - repo-backend
  - repo-docs
  git logアイコン

中央:
  「Agent Skill」
  - コミット分析
  - 機能別グルーピング
  - 重要度判定

  矢印「→」

右側（緑）「リリースノート」:
  ドキュメントアイコン

  「## v2.5.0」

  「新機能」
  - ダッシュボードウィジェット追加

  「改善」
  - ページ読み込み30%高速化

  「バグ修正」
  - セッションエラー修正

下部のメッセージ:
  「複数リポジトリの変更を統合レポートに」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 複数リポジトリを表現
- リリースノートのフォーマットを示す
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「フローチャート型」を採用：コミット→分析→リリースノートの流れ
- 複数リポジトリの統合を強調

### コンセプト
- 個人開発でも複数リポジトリの変更を追跡
- 機能ベースでグルーピングして分かりやすく
- リリースノートのフォーマット例を提示
