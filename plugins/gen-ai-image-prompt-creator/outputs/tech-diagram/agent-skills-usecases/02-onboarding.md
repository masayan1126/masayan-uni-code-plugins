# オンボーディング効率化 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】オンボーディング効率化（画像上部に大きく配置）

【図解タイプ】フローチャート（左から右への流れ）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青と緑のマーカー、白またはクリーム色の背景
【レイアウト】左から右への流れ、4ステップで表現

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、フローを指し示しながら解説している構図

【構成要素】
- ステップ1: 新メンバー参加（人アイコン）
- ステップ2: 自然言語で質問（吹き出しアイコン）
- ステップ3: Agent Skillが情報提供（ロボット/AIアイコン）
- ステップ4: スピーディにキャッチアップ（ロケットアイコン）
- 下部: 解決する課題3つを横並びで表示

【テキスト内容】
ステップ1: 「新メンバー参加」
  サブテキスト: 「何から始めれば...？」

ステップ2: 「自然言語で質問」
  サブテキスト: 「環境セットアップは？」「開発フローは？」

ステップ3: 「Skillが回答」
  サブテキスト: 「手順書・規約・過去の情報」

ステップ4: 「即キャッチアップ」
  サブテキスト: 「生産性UP」

下部（解決する課題）:
  ×「立ち上がり時間が長い」
  ×「同じ質問の繰り返し」
  ×「ドキュメントが見つからない」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- ステップ間は手書き風の矢印で接続
- 各ステップにシンプルなアイコン
- 下部の課題は×マークで強調
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「フローチャート型」を採用：参加→質問→回答→キャッチアップの流れを視覚化
- シンプルな4ステップで理解しやすく

### コンセプト
- 新メンバーの「何から始めれば？」という不安を解消
- 自然言語で質問できる手軽さを強調
- Before（課題）→After（解決）の対比
