# コードレビュー・品質チェック - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】コードレビュー・品質チェック（画像上部に大きく配置）

【図解タイプ】フローチャート（上から下への流れ）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青・黄・赤の信号機カラー、白またはクリーム色の背景
【レイアウト】上から下への流れ、7つの観点を放射状に配置

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、レビューフローを指し示しながら解説している構図

【構成要素】
- 上部: PRコード変更（ファイルアイコン）
- 中央: 7つのレビュー観点を円形に配置
- 下部: レビュー結果（3段階の重要度）

【テキスト内容】
上部:
  「PRコード変更」
  矢印で下へ

中央（7つの観点を円形に配置）:
  1. 「設計」（建物アイコン）
  2. 「セキュリティ」（鍵アイコン）
  3. 「パフォーマンス」（速度アイコン）
  4. 「可読性」（目アイコン）
  5. 「テスタビリティ」（チェックアイコン）
  6. 「保守性」（工具アイコン）
  7. 「ビジネス」（グラフアイコン）

下部（レビュー結果を3段階で）:
  🔴 Critical（要修正）: 「セキュリティ問題、バグ」
  🟡 Warning（推奨）: 「設計改善、リファクタ提案」
  🟢 Suggestion（任意）: 「可読性向上、命名改善」

右下のメッセージ:
  「多角的レビューで品質向上」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 7つの観点は放射状に配置
- 重要度は信号機カラー（赤・黄・緑）で色分け
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「フローチャート型」を採用：PR→観点チェック→結果の流れ
- 7つの観点を視覚的に網羅

### コンセプト
- レビューは多角的な観点が必要
- 指摘の重要度を3段階で分類
- 属人化を防ぎ、一貫性のあるレビューを実現
