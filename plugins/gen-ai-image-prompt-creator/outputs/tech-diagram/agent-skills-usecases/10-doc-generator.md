# ドキュメント・仕様書作成 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】ドキュメント・仕様書作成（画像上部に大きく配置）

【図解タイプ】概念図（変換プロセス）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青（入力）・緑（出力）のマーカー、白またはクリーム色の背景
【レイアウト】左にコード、中央に変換矢印、右に各種図解を配置

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、変換プロセスを指し示しながら解説している構図

【構成要素】
- 左側: ソースコード（ファイルアイコン群）
- 中央: Agent Skill（変換エンジン）
- 右側: 生成される図解・仕様書（4種類）

【テキスト内容】
左側（青）「ソースコード」:
  ファイルアイコン群
  - src/
  - services/
  - models/
  - api/

中央:
  「Agent Skill」
  大きな矢印「→」
  サブテキスト: 「自動生成」

右側（緑）4つの図解タイプ:
  1.「シーケンス図」
    - 処理の流れを可視化
    - 時系列で理解

  2.「クラス図」
    - クラス間の関係
    - 継承・依存関係

  3.「ER図」
    - DB設計
    - テーブル関係

  4.「API仕様書」
    - OpenAPI形式
    - エンドポイント一覧

下部のメッセージ:
  「コードとドキュメントの乖離を防ぐ」
  「実装から自動生成で常に最新」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- コード→図解の変換を大きな矢印で表現
- 4つの図解タイプをアイコンで区別
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「概念図型（変換プロセス）」を採用：入力→変換→出力の流れ
- 4種類の図解を出力として並列表示

### コンセプト
- コードから自動生成でドキュメントの乖離を防ぐ
- シーケンス図、クラス図、ER図、API仕様書の4種類
- 「常に最新」がキーメッセージ
