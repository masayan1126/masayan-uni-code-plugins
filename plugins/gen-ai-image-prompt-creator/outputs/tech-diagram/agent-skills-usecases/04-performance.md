# パフォーマンス最適化 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】パフォーマンス最適化（画像上部に大きく配置）

【図解タイプ】アーキテクチャ図（3層構造）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青（FE）・オレンジ（BE）・緑（DB）の3色マーカー、白またはクリーム色の背景
【レイアウト】上から下への3層構造、各層に最適化ポイントを表示

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、アーキテクチャを指し示しながら解説している構図

【構成要素】
- 上層: フロントエンド（ブラウザアイコン）- 青
- 中層: バックエンド（サーバーアイコン）- オレンジ
- 下層: データベース（DBアイコン）- 緑
- 各層に最適化ポイントを3-4個配置
- 層間を矢印で接続（データフロー）
- 左側: 「ボトルネック特定」の虫眼鏡アイコン

【テキスト内容】
上層（青）「フロントエンド」:
  最適化ポイント:
  - 「Core Web Vitals」（LCP, FID, CLS）
  - 「バンドルサイズ削減」
  - 「レンダリング最適化」
  - 「画像最適化」

中層（オレンジ）「バックエンド」:
  最適化ポイント:
  - 「API応答速度」
  - 「キャッシュ戦略」
  - 「非同期処理・並列化」
  - 「メモリ最適化」

下層（緑）「データベース」:
  最適化ポイント:
  - 「クエリ最適化」
  - 「インデックス設計」
  - 「N+1問題対策」
  - 「コネクション管理」

左側（虫眼鏡）:
  「ボトルネック特定」
  「計測→分析→改善」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 3層を明確に色分け
- 層間の矢印でデータフローを表現
- 各層に小さなアイコン（ブラウザ、サーバー、DB）
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「アーキテクチャ図型」を採用：FE/BE/DBの3層構造を視覚化
- 各層の最適化ポイントを網羅的に表示

### コンセプト
- パフォーマンスは3層すべてで対策が必要
- 「計測→分析→改善」のサイクルが重要
- ボトルネック特定が最初のステップ
