# フロントエンドデザイン - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】フロントエンドデザイン（画像上部に大きく配置）

【図解タイプ】比較表（Before/After）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】赤（Before/NG）・緑（After/OK）のマーカー、白またはクリーム色の背景
【レイアウト】左右2列でBefore/Afterを対比

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、比較を指し示しながら解説している構図

【構成要素】
- 左側（赤）: AIっぽいUI（Before）
- 右側（緑）: 高品質UI（After）
- 下部: スキルで避けるべきパターンと推奨パターン

【テキスト内容】
左側（赤）「AIっぽいUI」:
  サブタイトル: 「避けるべきパターン」

  ×マーク付きで:
  - 「グラデーション過多」
  - 「装飾過剰」
  - 「スタイル不統一」
  - 「ブランド不適合」

  簡易イラスト:
  - ごちゃごちゃしたUIのスケッチ
  - 派手なグラデーション

右側（緑）「高品質UI」:
  サブタイトル: 「推奨パターン」

  ○マーク付きで:
  - 「必要最小限の装飾」
  - 「一貫したスタイル」
  - 「WCAG準拠」
  - 「モバイルファースト」

  簡易イラスト:
  - クリーンでシンプルなUIのスケッチ
  - 余白を活かしたデザイン

下部のメッセージ:
  「Anthropic公式のフロントエンドデザインスキル」
  「たった1つのMarkdownファイルで実現」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- Before/Afterの対比を明確に
- 赤と緑で良い/悪いを色分け
- UIのスケッチは簡易的に
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「比較表型（Before/After）」を採用：AIっぽさの解消を視覚的に対比
- 良い/悪いを○×マークで明確化

### コンセプト
- AIが生成するUIの「AIっぽさ」問題を解決
- Anthropic公式のスキルを紹介
- 1ファイルで実現という驚き
