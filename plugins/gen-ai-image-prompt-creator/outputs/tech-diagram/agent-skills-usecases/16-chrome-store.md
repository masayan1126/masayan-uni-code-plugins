# Chromeストア申請チェック - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】Chromeストア申請チェック（画像上部に大きく配置）

【図解タイプ】フローチャート（左から右への流れ）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青（Chrome）・赤・緑のマーカー、白またはクリーム色の背景
【レイアウト】4ステップの申請フロー

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、申請フローを指し示しながら解説している構図

【構成要素】
- ステップ1: マニフェスト検証
- ステップ2: ポリシー準拠
- ステップ3: 素材準備
- ステップ4: 審査対応
- 下部: よくある却下理由

【テキスト内容】
ステップ1（青）「マニフェスト検証」:
  アイコン: ファイル
  - manifest.json
  - 必須フィールド
  - 権限の正当性
  - Manifest V3対応

ステップ2（青）「ポリシー準拠」:
  アイコン: チェックリスト
  - 禁止事項確認
  - ユーザーデータ取扱
  - 単一目的ポリシー
  - プライバシーポリシー

ステップ3（緑）「素材準備」:
  アイコン: 画像
  - アイコン（128x128）
  - スクリーンショット
  - 説明文
  - プロモーション素材

ステップ4（緑）「審査対応」:
  アイコン: 審査官
  - 審査通過
  - 却下対応
  - 再申請

下部（赤）「よくある却下理由」:
  ×「権限の過剰要求」
  ×「プライバシーポリシーなし」
  ×「V2のまま申請」

下部のメッセージ:
  「審査通過率を高める事前チェック」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- Chromeの青を意識
- 却下理由は×マークで警告
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「フローチャート型」を採用：申請までの4ステップを可視化
- よくある却下理由を警告として併記

### コンセプト
- 審査通過率を高めるための事前チェック
- Manifest V3対応を含む最新要件
- 却下理由を先回りして対策
