# セキュリティチェック - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】セキュリティチェック（画像上部に大きく配置）

【図解タイプ】概念図（中心から放射状）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】赤（警告）・青（対策）・グレー（背景）、白またはクリーム色の背景
【レイアウト】中心にOWASP Top 10、周囲に主要な脆弱性カテゴリを配置

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、セキュリティチェック項目を指し示しながら解説している構図

【構成要素】
- 中心: 盾アイコンと「OWASP Top 10」
- 周囲に5つの主要カテゴリ:
  1. インジェクション（注射器アイコン）
  2. 認証・認可（鍵アイコン）
  3. データ保護（ファイルアイコン）
  4. 入力検証（フィルターアイコン）
  5. 依存関係（パッケージアイコン）
- 各カテゴリに具体的なチェック項目

【テキスト内容】
中心:
  「OWASP Top 10」
  盾アイコン
  サブテキスト: 「体系的にチェック」

カテゴリ1（赤）「インジェクション」:
  - SQLインジェクション
  - XSS
  - コマンドインジェクション

カテゴリ2（青）「認証・認可」:
  - 認証チェック
  - 認可チェック
  - セッション管理

カテゴリ3（緑）「データ保護」:
  - 暗号化
  - シークレット管理
  - 個人情報保護

カテゴリ4（オレンジ）「入力検証」:
  - バリデーション
  - サニタイズ
  - ファイルアップロード

カテゴリ5（紫）「依存関係」:
  - 脆弱性スキャン
  - パッケージ更新
  - ライセンス確認

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 中心から放射状に5カテゴリを配置
- 各カテゴリを色分け
- 盾アイコンで「守る」イメージを強調
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「概念図型」を採用：OWASP Top 10を中心に主要観点を放射状に配置
- セキュリティの「守る」イメージを盾アイコンで表現

### コンセプト
- OWASP Top 10をベースに体系的なチェック
- 5つの主要カテゴリに整理して分かりやすく
- 「後付けのセキュリティ」を防ぐ
