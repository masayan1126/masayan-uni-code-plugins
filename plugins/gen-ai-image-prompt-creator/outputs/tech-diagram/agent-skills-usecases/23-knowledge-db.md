# ナレッジDB管理 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】ナレッジDB管理（画像上部に大きく配置）

【図解タイプ】概念図（循環サイクル）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青・オレンジ・緑・紫のマーカー、白またはクリーム色の背景
【レイアウト】4ステップの循環サイクル、中央にナレッジDB

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、ナレッジサイクルを指し示しながら解説している構図

【構成要素】
- 中央: ナレッジDB（データベースアイコン）
- 4つのステップ（循環）:
  1. 収集（Capture）
  2. 処理（Process）
  3. 整理（Organize）
  4. 活用（Use）
- 下部: 活用シーン

【テキスト内容】
中央:
  「ナレッジDB」
  データベースアイコン
  サブテキスト: 「情報を資産に」

ステップ1（青）「収集」:
  アイコン: ブックマーク
  - 記事・動画
  - 書籍・論文
  - ポッドキャスト
  - クイックキャプチャ

ステップ2（オレンジ）「処理」:
  アイコン: 歯車
  - 要約・抽出
  - ハイライト
  - タグ付け
  - 関連付け

ステップ3（緑）「整理」:
  アイコン: フォルダ
  - カテゴリ分類
  - Zettelkasten
  - PARA
  - Second Brain

ステップ4（紫）「活用」:
  アイコン: 出力矢印
  - プレゼン引用
  - ブログ記事
  - SNS発信
  - 意思決定

下部「活用シーン」:
  3つを横に並べる:
  - 「登壇資料の引用元」
  - 「ブログの参考材料」
  - 「SNSのネタ元」

下部のメッセージ:
  「情報を消費から資産に変える」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 4ステップを循環型で配置
- 中央にナレッジDBを配置
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「概念図型（循環サイクル）」を採用：収集→処理→整理→活用の継続的サイクル
- 知識管理手法（Zettelkasten等）を含む

### コンセプト
- 情報を「消費」から「資産」に変える
- 収集→処理→整理→活用のサイクル
- プレゼン、ブログ、SNSへの活用を強調
