# コーディングルール・詳細設計 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】コーディングルール・詳細設計（画像上部に大きく配置）

【図解タイプ】概念図（階層構造）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】青・紫・緑の3色マーカー、白またはクリーム色の背景
【レイアウト】上から下への階層構造、3つの柱を並列表示

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、構造図を指し示しながら解説している構図

【構成要素】
- 上部: Agent Skill（コーディングルール）を屋根のように配置
- 3つの柱（左から）:
  - 柱1: コーディング規約（青）
  - 柱2: デザインパターン（紫）
  - 柱3: SOLID原則（緑）
- 各柱の下に具体例を箇条書き
- 柱の間を点線で接続（相互関係）

【テキスト内容】
上部（屋根）:
  「Agent Skill: コーディングルール・詳細設計」
  サブテキスト: 「一貫したコード生成」

柱1（青）「コーディング規約」:
  - 命名規則
  - フォーマット
  - ファイル構成
  - エラーハンドリング

柱2（紫）「デザインパターン」:
  - Factory
  - Strategy
  - Observer
  - DI（依存性注入）

柱3（緑）「SOLID原則」:
  - 単一責任
  - 開放閉鎖
  - リスコフ置換
  - インターフェース分離
  - 依存性逆転

下部のメッセージ:
  「規約の属人化を排除し、一貫性のあるコードベースを実現」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 3つの柱を建物のように配置（基盤を支える構造）
- 各柱は色分けして区別
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「概念図型（階層構造）」を採用：3つの柱が基盤を支える構造で関係性を表現
- 建物の比喩で「支える」イメージを強調

### コンセプト
- コーディングルールは3つの柱で構成
- 相互に関連しながら一貫性のあるコードベースを支える
- 属人化の排除がキーメッセージ
