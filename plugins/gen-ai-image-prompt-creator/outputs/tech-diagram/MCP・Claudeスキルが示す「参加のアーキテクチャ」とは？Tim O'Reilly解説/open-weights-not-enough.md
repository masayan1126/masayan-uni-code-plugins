# オープンウェイトだけでは不十分 - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】オープンウェイトだけでは不十分（画像上部に大きく配置）

【図解タイプ】アーキテクチャ図（レイヤー構造）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】グレー（下層/オープンウェイト）とオレンジ（上層/インターフェース）の対比、白またはクリーム色の背景
【レイアウト】縦に2層構造、上層を大きく強調

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右側に配置、上層（インターフェース層）を指し示しながら「ここが本番！」という構図

【構成要素】
- 下層: オープンウェイト（モデルの重み）- グレー、小さめ
- 上層: インターフェース層（MCP・スキル）- オレンジ、大きく強調
- 矢印: 「本当のアクションはここ！」と上層を指す
- 人のアイコン: 上層に多数、下層に少数

【テキスト内容】
タイトル下サブテキスト: 「必要だが、十分ではない」

下層（グレー）「オープンウェイト」:
  - 「700億パラメータ」
  - 「プロセッサの命令のようなもの」
  - 「重要だが、普通の開発者が参加する場所ではない」
  アイコン: 重たい箱、南京錠が開いている
  人アイコン: 1〜2人（専門家のみ）

上層（オレンジ）「インターフェース層」:
  - 「MCP & スキル」
  - 「安定したわかりやすいインターフェース」
  - 「問題を理解し、解決策を想像し、説明する」
  アイコン: プラグ、ドキュメント、ツール
  人アイコン: 多数（開発者＋非開発者）

  強調テキスト（大きく）: 「本当のアクションはここ！」

下部の補足:
  「プログラミング ≠ 特定言語を使うこと」
  「= 問題を理解し、ツールに説明するスキル」

右側の吹き出し:
  「ChatGPTのシステムプロンプトも実はこのパターン」
  （手書きの波線で囲む）

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 下層は控えめに、上層を大きく目立たせる
- 人のアイコンで「参加者の数」を視覚化
- 矢印で「ここが重要」を強調
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「アーキテクチャ図型（レイヤー構造）」を採用
- 下層と上層の関係を視覚的に表現
- 「本当のアクションはどこか」を強調

### コンセプト
- O'Reillyの洞察：オープンウェイトは「必要だが十分ではない」
- 参加のアーキテクチャの観点：インターフェース層こそが参加の場
- プログラミングの再定義：言語ではなくスキルセット

### キーワード
- **オープンウェイト**: 重要だが、専門家向け
- **インターフェース層**: 本当の参加の場、民主化
- **システムプロンプト**: 暗黙のパターンの明示化

### シリーズとしての位置づけ
1. 「参加のアーキテクチャ」- 4つの特性を定義
2. 「MCPとスキル＝ソースを表示」- Web時代との対比
3. 「GPT vs スキル」- 2つのモデルの違い
4. 「オープンウェイトだけでは不十分」- どこで参加するか ← 今回
