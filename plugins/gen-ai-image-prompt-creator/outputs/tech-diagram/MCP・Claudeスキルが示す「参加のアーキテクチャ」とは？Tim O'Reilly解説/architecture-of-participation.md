# 参加のアーキテクチャ - 図解プロンプト

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】参加のアーキテクチャ（画像上部に大きく配置）

【図解タイプ】概念図
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】マーカーペン風の4色（青・緑・オレンジ・ピンク）、白またはクリーム色の背景
【レイアウト】中心から放射状に4つの要素を配置

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を右下に配置、ホワイトボードを指し示しながら解説している構図

【構成要素】
- 中心: 「参加のアーキテクチャ」（大きな丸で囲む）
- 要素1: 可読性（Readability）- 青色マーカー
- 要素2: 変更可能性（Modifiability）- 緑色マーカー
- 要素3: 構成可能性（Composability）- オレンジ色マーカー
- 要素4: 共有可能性（Shareability）- ピンク色マーカー
- 各要素は手書きの線で中心と接続

【テキスト内容】
中心: 「参加のアーキテクチャ」
  サブテキスト: 「オープンソースが変革をもたらした理由」

要素1（青）: 「可読性」
  説明: 「全体を理解しなくても、部分が読める」
  例: 本のアイコン、虫眼鏡

要素2（緑）: 「変更可能性」
  説明: 「1行変えるだけで挙動が変わる」
  例: 設定ファイル、ペンで書き換えるイメージ

要素3（オレンジ）: 「構成可能性」
  説明: 「小さな部品を組み合わせて複雑なことができる」
  例: パイプで繋がるブロック、レゴ風

要素4（ピンク）: 「共有可能性」
  説明: 「便利な関数1つでも価値がある」
  例: シェアアイコン、手渡しのイメージ

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き
- ホワイトボードに描いたような質感
- マーカーペンのタッチ（線に少しブレがある）
- 親しみやすい雰囲気
- 日本語テキスト
- 各要素に小さな手書きアイコンを添える
- 矢印や接続線も手書き風
- 余白を十分に確保
- 男の子とインコが一緒に解説している構図
```

## 設計メモ

### 図解タイプの選定理由
- 「概念図型」を採用：4つの抽象概念を視覚的に表現するため
- 中心から放射状に配置することで、4つの特性の対等性を表現

### カラー選定理由
- 手書き風テイストに合わせ、マーカーペン風の鮮やかな4色を使用
- 各特性を色で区別し、一目で違いが分かるように

### コンセプト
- O'Reillyの「参加のアーキテクチャ」の本質を視覚化
- 「コードが無料だから」ではなく「参加しやすいから」という洞察を強調
- 手書き風のカジュアルな雰囲気で、難しい概念を親しみやすく
