# todoリストとLost in the Middle問題

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
【タイトル】todoリストが解決する「Lost in the Middle」問題（画像上部に大きく配置）

【図解タイプ】概念図
【テイスト】手書き風: 手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】AI・機械学習系（青・紫ベース）、ダークテーマ
【レイアウト】上下2段構成（Before/After）、中央に矢印で変化を示す

【解説キャラクター】添付画像のキャラクター（男の子とインコ）を右下に配置、図解を指し示すポーズ、吹き出しで「これで忘れない！」

【構成要素】
- コンテキストウィンドウ: 横長のバーで表現（先頭・中間・末尾の3セクション）
- Attention分布: グラデーションで視覚化（先頭と末尾は濃い色、中間は薄い色）
- todoリスト更新: 矢印で「目標」が中間から末尾へ移動する様子
- Before/After: 上段が問題状態、下段が解決状態

【テキスト内容】
上段（Before - 問題）:
- コンテキストバー: 左端「システムプロンプト」、中央「目標が埋もれる...」（薄い色で表現）、右端「直近のやり取り」
- Attention分布の説明: 「先頭と末尾に注意が集中、中間は見落とされがち」

下段（After - 解決）:
- コンテキストバー: 左端「システムプロンプト」、中央「過去のやり取り」、右端「todoリスト更新！目標がここに」（明るく強調）
- 矢印と説明: 「todoを更新するたび目標がコンテキスト末尾に再配置」

中央の変化を示す矢印: 「todoリスト更新」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような
- マーカーペンのタッチ、親しみやすい雰囲気
- 日本語テキスト
- 余白を十分に確保
- Attention分布をグラデーションで直感的に表現
- Before/Afterの対比を明確に
- 「目標」を星マークやハイライトで強調
- 添付画像のキャラクター（男の子とインコ）が解説している構図
```
