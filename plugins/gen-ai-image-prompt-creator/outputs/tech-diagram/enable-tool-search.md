# ENABLE_TOOL_SEARCH=true 図解

## プロンプト

```
【タイトル】ENABLE_TOOL_SEARCH=trueでトークン74k削減（画像上部に大きく配置）

【図解タイプ】比較表（Before/After）
【テイスト】手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
【カラースキーム】赤（Before/問題）と緑（After/解決）のコントラスト、白背景
【レイアウト】左右2分割、Before（左）とAfter（右）の対比

【解説キャラクター】添付画像のキャラクター（男の子とインコたち）を下部中央に配置、両手を広げてBeforeとAfterを示すポーズ

【構成要素】
- Before（左・赤系）: 従来方式の問題点
- After（右・緑系）: ツール検索有効化後
- 中央: 変化を示す矢印と「環境変数1つで解決」のテキスト
- 下部: 効果のサマリー

【テキスト内容】
タイトル: 「ENABLE_TOOL_SEARCH=trueでトークン74k削減」

Before（左側・赤マーカー）:
- 「MCPツール定義」
- 「74.5kトークン消費」
- 「コンテキストの37%を占有」
- 「Context rot（コンテキストの腐敗）」
- バツ印アイコン
- 円グラフ: 37%が赤く塗られている

After（右側・緑マーカー）:
- 「オンデマンドロード」
- 「トークン消費 0k」
- 「Free space 134k（66.9%）」
- 「精度向上 79.5%→88.1%」
- 丸印アイコン
- 円グラフ: ほぼ全体が緑（空き容量大）

中央の矢印部分:
- 「ENABLE_TOOL_SEARCH=true」
- 「環境変数1つ」
- 大きな右向き矢印

下部サマリー:
- 「使い方: ENABLE_TOOL_SEARCH=true claude」

【スタイル指示】
- 手書き風スケッチ、ラフな走り書き、ホワイトボードに描いたような、マーカーペンのタッチ、親しみやすい雰囲気
- 日本語テキスト
- BeforeとAfterで色分け（赤 vs 緑）を明確に
- 手書きの円グラフでコンテキスト使用状況を視覚化
- バツ印と丸印で直感的に良し悪しを表現
- 余白を十分に確保
- 添付画像のキャラクター（男の子とインコ）が解説している構図
```

## 図解のポイント

- **Before/After比較**: 視覚的に問題と解決策の差を明確化
- **数値の強調**: 74k削減、37%→0%、精度向上の具体的数値
- **シンプルな解決策**: 「環境変数1つ」という手軽さを強調
- **手書き風**: 技術的内容を親しみやすく伝える

## 使用時の注意

プロンプトを画像AIに入力する際は、必ずキャラクターの元画像（男の子とインコ）を一緒に添付してください。
