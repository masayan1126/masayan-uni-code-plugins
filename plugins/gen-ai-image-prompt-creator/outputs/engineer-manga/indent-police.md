# インデント警察

> **重要**: このプロンプトを画像AIに入力する際は、キャラクターの元画像を一緒に添付してください。

## プロンプト

```
# 1. 役割と目標設定 (Role & Goal)
あなたは日本のプロ漫画家兼ストーリーボード演出家です。

以下のシナリオに基づき、3コマ構成のエンジニアあるある漫画を、1枚の高品質なカラー画像として描写してください。

【タイトル】インデント警察（画像上部に大きく配置）

# 2. レイアウトとフォーマット (Layout & Format)
【形式・レイアウト】3コマ、縦配置、表情にフォーカス
【読み順】上から下へ
【枠線】黒い枠線と白い余白（solid black panel borders and white gutters）

# 3. キャラクター設定 (Character Bible) ※外観の一貫性が最重要
添付画像のキャラクター（男の子とインコたち）を使用。
Use the attached character image as the reference for appearance, proportions, coloring, and emotional style.
Maintain strict character consistency across all panels.
外観・プロポーション・配色・表情のスタイルを全コマで厳密に一貫させてください。

【役割】主人公はコードレビューを受ける側。インコはリアクション担当。

【背景設定】オフィスのデスク、モニター画面にコードが表示されている

# 4. シナリオ (Scenario)
## コマ1
- 場面設定: オフィス・コードレビュー中
- カメラ: ミディアム
- 描写: モニターを見せながら説明する主人公。自信満々の表情。画面にはコードが表示されている。インコが肩に乗っている。
- セリフ: 「新機能の実装できました！レビューお願いします！」
- 効果音: なし

## コマ2
- 場面設定: 同じオフィス
- カメラ: クローズアップ
- 描写: レビュアー（画面外からの吹き出し）のコメントを見て固まる主人公。インコも一緒に固まっている。モニターには赤い指摘コメントが表示されている。
- セリフ: レビュアー「23行目、インデントがスペース4つになってます。うちは2つです」
- 効果音: ピシッ

## コマ3
- 場面設定: 同じオフィス
- カメラ: 大アップ
- 描写: 虚無の表情の主人公。背景には大量の赤い指摘コメント（すべてインデント関連）が浮かんでいる。インコは白目。吹き出しには追加の指摘が並ぶ。
- セリフ: レビュアー「45行目も」「78行目も」「あとここも」 / 主人公（心の声）「ロジックは...見てくれないの...？」
- 効果音: シーン...

# 5. 演出とスタイル (Direction & Style)
【ジャンル・雰囲気】ギャグ、一発ネタ、共感系
【画風】カラー、手書き風、シンプル、デフォルメ、表情豊か
【線画】シンプルで柔らかい線、スケッチ調のタッチ
【演出効果】表情の変化を強調（自信満々→固まる→虚無）、コマ3は背景に赤いコメントを散らして圧迫感を演出
```

## 制作メモ

- パターン: キャラ固定ギャグ（3コマ）
- ポイント: 表情の変化（自信満々→固まる→虚無）がオチの肝
- 共感ポイント: 本質的なロジックよりフォーマットを指摘される経験
